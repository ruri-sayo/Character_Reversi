# 不具合調査と対策報告書

## 1. 発生している現象
キャラクターを追加した結果、画面内の要素が増えたことで全体の高さが画面の高さ（ビューポート）を超過しました。
現在、親コンテナの設定が「画面中央揃え（`justify-center`）」かつ「画面高さいっぱい（`h-screen`）」に固定されているため、中身が画面より大きくなった際に**上方向へのオーバーフロー**が発生しています。
CSSの仕様上、中央揃えの状態でオーバーフローが発生すると、開始位置（上端）が画面外に見切れてしまい、スクロールしても到達できなくなる現象（クリッピング）が発生します。これにより、上部にある「先行・後攻スイッチ」が操作不能になっています。

## 2. 原因の特定
`index.html` の `body` タグおよび内部コンテナのクラス設定に起因します。

**該当コード:**
```html
<body class="bg-slate-100 text-slate-800 h-screen flex flex-col items-center justify-center relative">
```
*   `h-screen`: 高さを画面サイズ（100vh）に固定している。
*   `justify-center`: フレックスボックスで垂直方向の中央揃えを行っている。

この組み合わせは、コンテンツが画面内に収まる場合は美しく中央配置されますが、コンテンツが画面サイズを超えると、コンテンツの中心を画面の中心に合わせようとするため、上部と下部の両方にはみ出しが発生します。通常、ブラウザのスクロールは「上から下」へ行われるため、上にはみ出した部分は操作不能になります。

## 3. 対策
レイアウトの基本方針を「コンテンツが少ないときは中央、多いときはスクロール可能」な形に変更します。

**修正案:**
1.  `h-screen`（高さ固定）を `min-h-screen`（最小高さ画面サイズ）に変更します。これにより、コンテンツが増えた際に画面エリア自体が縦に伸びるようになります。
2.  `justify-center` を削除し、代わりにコンテナ側でマージン調整（`my-auto` 等）を行うか、パディング（`py-10`）等で適切な余白を確保します。今回は `my-auto` の挙動を活かしつつ、安全にスクロール可能にするために `py-8`（上下余白）を追加し、`relative` な配置を維持します。

**修正後のイメージ:**
```html
<body class="bg-slate-100 text-slate-800 min-h-screen flex flex-col items-center py-8 relative">
```

## 4. 実施結果
`index.html` に対して上記の修正を適用しました。これにより、キャラクターが増えても画面全体のスクロールが可能になり、上部のスイッチが見切れることなく操作可能になります。
