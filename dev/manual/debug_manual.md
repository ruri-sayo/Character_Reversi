# デバッグモード操作マニュアル

## 概要
開発中の動作確認や特定の状況（終盤、パスなど）の再現を容易にするために、デバッグモードを実装しました。この機能は開発者向けであり、通常のゲームプレイでは表示されません。

## 1. デバッグモードの起動方法

1.  ブラウザでゲームを開きます。
2.  開発者ツール（Console）を開きます（F12キー または Ctrl+Shift+I）。
3.  以下のコマンドを実行します。

```javascript
enableDebug()
```

成功すると、画面右上に **"DEBUG MODE ON"** という赤い表示が出現します。

## 2. デバッグ機能一覧

### 2.1. 盤面の強制ロード
特定の盤面パターンをロードして、即座にその状況から開始できます。

**コマンド:**
```javascript
loadDebugBoard('パターン名')
```

**使用可能なパターン:**
- `'endgame_win'`: 終盤、黒（あなた）が勝ち確定の状況
- `'endgame_lose'`: 終盤、白（AI）が勝ち確定の状況
- `'pass_test'`: 黒が打つ場所がない（パスが発生する）状況

例：
```javascript
loadDebugBoard('endgame_win')
```

### 2.2. マスの強制変更
盤面上のマスを **Shiftキーを押しながらクリック** すると、そのマスの状態を強制的に変更できます。

クリックするたびに以下の順序で変化します：
- 空き(0) → 黒(1) → 白(-1) → 空き(0) ...

これにより、自由な盤面を作成してテストすることが可能です。

## 3. デバッグパターンの追加方法

新しいテストパターンを追加したい場合は、以下のファイルを編集してください。

**ファイルパス:** `dev/debug/debug.json`

このJSONファイルの `patterns` 配列に新しいオブジェクトを追加します。

**フォーマット:**
```json
{
  "name": "呼び出し用ID（英数字）",
  "description": "パターンの説明",
  "turn": 1, // 1:黒の手番, -1:白の手番
  "board": [
    // 8x8の盤面配列 (1:黒, -1:白, 0:空き)
    [0, 0, 0, ...], 
    ...
  ]
}
```

追加後、ページをリロードすると新しいパターン名で `loadDebugBoard()` が使えるようになります。
